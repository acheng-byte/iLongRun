# 发版说明 v0.8.1 — `ilongrun-model` 交互式选模器与 `/ilongrun*` 中文菜单说明

## 概述

`v0.8.1` 是对 `v0.8.0` 的一次交互体验修正：

- `ilongrun-model` 不再作为 Copilot CLI 会话内的 `/ilongrun-model` skill 暴露
- `ilongrun-model` 改成**终端裸命令**的交互式选模器
- `/ilongrun`、`/ilongrun-prompt`、`/ilongrun-resume`、`/ilongrun-status` 的菜单说明统一改成中文

这次更新不会改变 `--model` 的全链路锁模语义，也不会改变 review / final audit 固定模型策略。

## 新增

### 1. `ilongrun-model` 现在支持交互式选模

直接在终端运行：

```bash
ilongrun-model
```

行为如下：

- **TTY 终端**：进入交互式 picker
- **非 TTY**：默认退化为 `show`

同时新增：

```bash
ilongrun-model show
ilongrun-model --refresh
ilongrun-model claude-haiku-4.5
ilongrun-model reset
```

其中：

- `show`：查看当前模板
- `--refresh`：先刷新模型缓存，再进入 picker / show
- `<slug-or-alias>`：直接设置模型
- `reset`：恢复模板默认值

### 2. 会话菜单说明中文化

现在 Copilot CLI 会话菜单里保留这些中文说明入口：

- `/ilongrun`
- `/ilongrun-prompt`
- `/ilongrun-resume`
- `/ilongrun-status`

## 增强

### 1. `ilongrun-model` 职责更清晰

它现在只负责修改 **iLongRun 默认模板**：

- 会改：`run / coding / ilongrun / ilongrun-coding` 与主执行角色模板
- 不会改：
  - `codingAuditModel`
  - `roleModels.final-audit-reviewer`
  - review 固定角色
  - `prompt / resume / status / doctor`
  - `fallback / backoff / aliases / displayNames`

同时它**不会**改变当前 Copilot CLI 会话里的原生 `/model`。

### 2. 安装与 doctor 闭环更新

- 安装升级时会主动清理旧的 `~/.copilot/skills/ilongrun-model`
- doctor 不再把 `/ilongrun-model` skill 当成缺失项报错
- 如果检测到旧残留，只会给出 legacy warn，提醒清理

### 3. skill lint 放宽为中英文 trigger-first

现在 frontmatter `description` 接受：

- `Use when ...`
- `当用户...时使用`
- `当需要...时使用`

## 说明

### 关于 `/ilongrun-model`

当前 GitHub Copilot CLI 的公开扩展能力不支持把自定义命令注册成新的原生 `/model` 式弹窗。

因此本次采用的方案是：

- 会话内移除 `/ilongrun-model`
- 终端侧提供 `ilongrun-model` 自建交互式选模器

这是当前可实现能力下，更贴近原生体验、同时又不污染会话模型语义的方案。

## 安装升级

建议直接重装一次，确保旧 skill 残留被清理：

```bash
curl -fsSL https://raw.githubusercontent.com/izscc/iLongRun/main/install.sh | bash
```

安装后建议验证：

```bash
command -v ilongrun-model
ilongrun-model
python3 scripts/lint_ilongrun_skills.py
python3 scripts/selftest_ilongrun.py
bash scripts/ilongrun-doctor --refresh-model-cache
```
