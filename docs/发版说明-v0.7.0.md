# 发版说明 v0.7.0 — `ilongrun-coding` 升级为 Coding Discipline Kernel

## 概述

v0.7.0 的重点，不是再给 `ilongrun-coding` 补几条纪律，而是把它从 **Coding Swarm Protocol** 继续升级成一个带 Superpowers 方法学强化层的 **Coding Discipline Kernel**。

这次升级保持了原有蜂群骨架：

- `serial / swarm-wave / super-swarm`
- `phase-define → phase-plan → phase-build → phase-verify → phase-review → phase-audit → phase-finalize`

但把以下方法学正式写进协议与账本：

- workspace isolation
- task microcycle
- claim verification
- root-cause recovery
- skill engineering

## 这次改了什么

### 1. coding 协议真值升级到 0.7.0

`config/coding-protocol.jsonc` 新增：

- `methodologyOverlay`
- `workspaceIsolationPolicy`
- `taskMicrocycle`
- `claimVerificationPolicy`
- `debugPolicy`
- `skillEngineeringPolicy`

### 2. scheduler / workstream 账本升级

scheduler 新增：

- `workspaceIsolation`
- `phaseGuards`
- `claimVerification`

workstream 新增：

- `specRef`
- `microcycleState`
- `reviewSequence`
- `freshEvidence`
- `rootCauseRecord`

### 3. finalize / recovery 变成硬门禁

- finalize 缺 fresh evidence 时直接 blocked
- failed / blocked workstream 缺 root cause record 时直接 blocked
- build workstream 若 microcycle / reviewSequence 未收敛，不得算完整完成

### 4. skills 进入工程化治理

- 顶层 skill frontmatter 改为 `Use when...`
- `ilongrun-coding` 新增 5 个方法学 playbooks
- 新增 `lint_ilongrun_skills.py`
- 新增 pressure-scenario 参考资料

### 5. 看板与安装链路同步升级

- `ilongrun-status` 新增方法学门禁展示
- `ilongrun-doctor` 新增 protocol 字段 / playbooks / skill lint 检查
- 安装器同步安装新的 playbooks、references、helper
- 安装看板继续显示当前安装版本号

## 升级建议

### 对已有用户

1. 重新执行一键安装，确保新的 helper / playbooks / references 全部落地
2. 执行：

```bash
ilongrun-doctor --refresh-model-cache
```

3. 若恢复旧 run，注意 0.7.0 是 **新协议优先**：
   - 旧 run 可以 best-effort 读取与 reconcile
   - 但缺 `freshEvidence` / `rootCauseRecord` / `reviewSequence` 时，不会被静默放行

### 对新 run

推荐直接从新的 coding run 开始体验：

```bash
ilongrun-coding "实现一个 TypeScript 模块，补测试，完成 review gate 与最终终审"
```

然后查看：

```bash
ilongrun-status latest
```

重点看：

- 方法学门禁
- workspace isolation
- claim verification
- reviewSequence / microcycle 是否收敛

## 核心结论

v0.7.0 不是把 iLongRun 推倒重来。

它做的是：

> **保留原有蜂群编排骨架，把 coding 执行过程升级成更科学、更严格、更可验证的方法学内核。**

所以这次升级的本质是：

- 蜂群骨架不变
- 编码纪律增强
- finalize / recovery 更严格
- skills 自身也进入工程化治理
