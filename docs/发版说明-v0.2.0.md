# 发版说明 v0.2.0 — 编码纪律升级

> 发版日期：2026-04-06

## 概述

v0.2.0 是 iLongRun 的第一次重大功能升级。核心目标：**让 iLongRun 在 coding 任务场景下具备完整、科学、可执行的编码纪律**。

灵感来源：[addyosmani/agent-skills](https://github.com/addyosmani/agent-skills) — 一个面向 AI 代理的生产级工程技能包。

## 什么变了

### 新增：`ilongrun-coding` 技能

编码纪律的**单一真值源**。当 `profile=coding` 时自动生效。

涵盖七大编码纪律：
1. **TDD 纪律** — Red-Green-Refactor 循环 + 测试金字塔 + Prove-It 模式
2. **增量实现纪律** — 薄垂直切片 + 原子提交 + 范围纪律
3. **系统化调试纪律** — Stop-the-Line 规则 + 五步排查法
4. **代码审查纪律** — 五轴审查框架 + 发现分级
5. **安全加固纪律** — 三层边界系统 + OWASP Top 10
6. **性能优化纪律** — 先测量后优化 + Core Web Vitals
7. **Git 工作流纪律** — 原子提交 + 描述性消息

### 新增：`ilongrun-code-reviewer` 代理

五轴代码审查专家，可被 Mission Governor 调用。

### 新增：参考清单

- `references/testing-patterns.md` — 测试模式速查
- `references/security-checklist.md` — 安全清单速查
- `references/performance-checklist.md` — 性能优化清单速查

### 增强：所有现有代理

每个代理都整合了相应的编码纪律能力：

| 代理 | 增强内容 |
|------|----------|
| Executor | TDD 循环 + Stop-the-Line + 增量实现 |
| GPT-5.4 Reviewer | 五轴审查 + 参考清单集成 |
| Mission Governor | 编码生命周期 + 借口防御 + Adjudication |
| Strategy Synthesizer | 任务画像增强 + 垂直切片 + 大小估算 |
| Phase Planner | 编码 Phase 映射 + Wave 资格评估 |
| Workstream Planner | 大小估算 + 垂直切片 + 模板 |
| Recovery Agent | 系统化调试 + 恢复场景表 |

### 增强：主 skill

- 编码生命周期映射（DEFINE→PLAN→BUILD→VERIFY→REVIEW→SHIP）
- 合理化借口防御
- Red Flags 危险信号
- Verification Checklist

## 升级方法

```bash
# 如果已安装 v0.1.0
curl -fsSL https://raw.githubusercontent.com/izscc/iLongRun/main/install.sh | bash
```

安装脚本会自动备份旧文件并覆盖安装。

## 兼容性

- 完全向后兼容 v0.1.0
- 新增文件不影响既有行为
- `ilongrun-coding` 技能仅在 `profile=coding` 时生效
- `ilongrun-code-reviewer` 代理为可选调用

## 已知限制

- 编码纪律目前为指导性规范，尚未硬编码为 scheduler phase/gate
- Code Reviewer 代理需要手动调用（自动调用路由将在 v0.3.0 实现）
- 参考清单以 Web 应用场景为主，其他领域需自行补充
