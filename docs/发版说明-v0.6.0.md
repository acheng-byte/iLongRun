# 发版说明 v0.6.0 — `ilongrun-coding` 深度升级为 Coding Swarm Protocol

> 发版日期：2026-04-07

## 概述

v0.6.0 的重点，不是再给 `ilongrun-coding` 补几条纪律，而是把它正式升级成一个 **可长跑、可恢复、可审计、可发布的 Coding Swarm Protocol**。

这一版把三条线真正合在了一起：

1. **iLongRun 现有的长运行账本内核**：`scheduler.json + workstreams/*/status.json`
2. **Swarm 波次编排模型**：依赖图、wave 划分、协调器记忆、并行/串行切换
3. **agent-skills 工程方法学**：spec、planning、incremental implementation、TDD、debugging、review、security、shipping

最终目标是让 coding mission 从：

> “一个带纪律文案的 prompt”

升级为：

> “一个带协议真值、阶段 playbooks、质量门禁、恢复语义和安装/doctor/状态链路支持的长运行编码内核”。

## 这次改了什么

### 1) 新增 coding protocol 机器真值

新增：

- `config/coding-protocol.jsonc`

它现在负责统一描述：

- phase 定义：`define / plan / build / verify / review / audit / finalize`
- swarm policy：`serial / swarm-wave / super-swarm`
- JS/TS 优先语言画像
- review matrix：`code / test-evidence / security`
- release policy：哪些条件下必须阻断 finalize

这意味着 coding 规则不再散落在多个 Markdown 里，而是第一次拥有了**脚本与 skill 共用的机器真值源**。

### 2) `ilongrun-coding` 从“大总纲”重构成“元技能 + phase playbooks”

保留：

- `skills/ilongrun-coding/SKILL.md`

但它现在只负责：

- 协议路由
- 生命周期切换
- swarm 规则
- Copilot CLI 运行约束

细节则下沉到新的内部 playbooks：

- `phase-define.md`
- `phase-plan.md`
- `phase-build.md`
- `phase-verify.md`
- `phase-review.md`
- `phase-ship.md`
- `swarm-policy.md`
- `js-ts-profile.md`

这样 `ilongrun-coding` 不再是“大而全的纪律说明书”，而是**可调度、可维护、可扩展的协议入口**。

### 3) coding 生命周期正式接入 scheduler

从这一版开始，`profile=coding` 不再复用泛化的 `phase-execution`。

新的 coding run 固定采用：

```text
phase-define
→ phase-plan
→ phase-build
→ phase-verify
→ phase-review
→ phase-audit
→ phase-finalize
```

同时 scheduler / workstream 协议新增字段：

- scheduler
  - `codingProtocol`
  - `swarmPolicy`
  - `dependencyGraph`
  - `reviewMatrix`
- workstream
  - `skillPack`
  - `swarmMode`
  - `writeSet`
  - `handoffArtifacts`
  - `entryCriteria`
  - `exitCriteria`
  - `verificationClass`
  - `reviewRequired`

这让 coding run 的结构首次变成**显式、可审计、可恢复**的账本对象，而不再只是靠 prompt 约定。

### 4) build 波次支持真正的 coding swarm

v0.6.0 把 swarm 策略真正落到了编码编排上：

- build 切片 workstream 会显式生成 write set / handoff / acceptance / verify 合同
- 共享上游依赖但彼此不互相依赖的 build wave，现在可以合法进入 `/fleet` 评估
- `review / audit / finalize / git / release` 一律 internal

也就是说：

- **可并行的 build** 可以并行
- **必须串行的 gate** 会被强制串行

### 5) 独立 `phase-review` gate 成为 finalize 前硬门禁

新增专项 review agents：

- `agents/ilongrun-test-engineer.agent.md`
- `agents/ilongrun-security-auditor.agent.md`

并保留既有：

- `ilongrun-code-reviewer.agent.md`
- `ilongrun-gpt54-audit-reviewer.agent.md`

新的 `phase-review` 默认包含三条 gate：

- `review-code`
- `review-test-evidence`
- `review-security`

规则也正式硬化：

- 缺任一 review gate，不得 finalize
- `phase-audit` 不能替代 `phase-review`
- `pendingMustFixCount > 0` 时，依旧阻断 finalize

### 6) strategy / plan / task-list / brief 全部升级为协议投影

从这一版开始，投影文件不再只讲“做什么”，还会讲：

- 当前 coding protocol 版本
- swarm active mode
- dependency graph 节点/边数量
- workstream 的 swarm mode / write set / handoff artifacts
- review matrix gate 状态
- worker mini-contract（inputs / outputs / entry / exit / skill pack）

这让：

- `strategy.md`
- `plan.md`
- `task-list-N.md`
- `workstreams/*/brief.md`

真正成为**协议层投影**，而不是普通说明文档。

### 7) doctor / install / status / launch 全链路理解新协议

这次同时补齐了非执行链路：

- `render_ilongrun_status_board.py`
  - 展示 coding protocol、swarm、review matrix、gate 状态
- `render_ilongrun_launch_board.py`
  - 展示新 phase 与 coding protocol 版本
- `copilot-ilongrun doctor`
  - 新增 protocol bundle 检查：
    - `coding-protocol.jsonc`
    - `vendor/agent-skills`
    - `ilongrun-coding` playbooks
    - 专项 review agents
- 安装/卸载脚本
  - 新增安装 `coding-protocol.jsonc`
  - 新增安装 `vendor/agent-skills/`

### 8) 引入 vendorized `agent-skills` 快照

仓库新增：

- `vendor/agent-skills/`

本次只固化实际采用的精选技能快照，并带上 LICENSE 与来源说明。

这样做的目的不是把上游原样镜像进来，而是：

- **保留可追溯来源**
- **运行时只依赖 iLongRun 自己的适配协议**
- 避免线上仓库变动直接影响本地行为

## 升级建议

```bash
curl -fsSL https://raw.githubusercontent.com/izscc/iLongRun/main/install.sh | bash
ilongrun-doctor --refresh-model-cache
```

如果你已经安装过旧版，建议重点确认：

```bash
ilongrun-doctor
ilongrun-status latest
```

尤其是 coding 用户，建议确认 doctor 里以下四项都正常：

- Coding 协议
- Vendor 快照
- Skill Playbooks
- 专项 Agents
