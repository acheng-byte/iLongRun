# 发版说明 v0.8.0 — `ilongrun-model` 主模型热切换、审查模板收敛与选模链路修正

## 概述

`v0.8.0` 这次的重点，是把“默认模型怎么改、改完到底影响到哪里”这件事做成正式产品能力。

你现在可以直接用：

```bash
ilongrun-model
ilongrun-model claude-haiku-4.5
ilongrun-model reset
```

来热切换 **后续新任务** 的默认主模型模板，而不必每次都手写 `--model`。

## 新增

### 1. 新增 `ilongrun-model`

本次新增三套统一入口：

- Shell：`ilongrun-model`
- 高级入口：`copilot-ilongrun model`
- Copilot CLI 对话框：`/ilongrun-model`

支持三种操作：

- `show`：查看当前主模型模板与写入目标
- `set`：切换默认主模型
- `reset`：恢复模板默认值

### 2. 新增模型管理 helper

新增：

- `scripts/manage_ilongrun_model.py`

负责：

- alias 归一化
- 未知模型报错
- 双写安装态 / 仓库态配置
- `show / set / reset` 输出

## 增强

### 1. 主执行模板可热切换

`ilongrun-model <slug>` 会更新：

- `commandDefaults.run`
- `commandDefaults.coding`
- `skillDefaults.ilongrun`
- `skillDefaults.ilongrun-coding`
- 主执行/规划/恢复相关角色模板

但它只影响：

- **后续新启动** 的 `ilongrun`
- **后续新启动** 的 `ilongrun-coding`

不会回写：

- 当前 active run 的 `scheduler.json`
- 正在执行中的 mission 账本

### 2. resume 默认继承历史 run 模型

如果没有显式传 `--model`，`resume` 现在会优先读取目标 run 的：

- `selectedModel`
- `fallbackChain`

因此：

- 新默认模板不会打断旧 run 的继续执行
- `ilongrun-model` 更适合作为“未来任务模板切换器”

### 3. 审查模板收敛为 GPT-5.4

本次把默认审查模板统一收敛为：

- `code-reviewer` → `gpt-5.4`
- `test-engineer` → `gpt-5.4`
- `security-auditor` → `gpt-5.4`

同时保持这些项**不被 `ilongrun-model` 改写**：

- `codingAuditModel`
- `roleModels.final-audit-reviewer`
- `prompt / resume / status / doctor` 默认模型
- `fallback / backoff / aliases / displayNames`

> 说明：本版本只固化 model slug，不新增 `reasoning_effort` 真值字段；`xhigh` 仍作为文档约定，不进入配置协议。

### 4. launcher / supervisor 选模链路修正

本次顺手修复了一个容易让人误判的问题：

- 之前 launcher 预选模型和 supervisor 首跳模型可能不一致
- `coding` 首跳有机会被误算到 fallback 首项

现在统一为：

- `run` 首跳按 run 模板
- `coding` 首跳按 coding 模板
- `resume` 首跳优先继承历史 run

## 安装升级

安装/卸载链路已补齐：

- `ilongrun-model` launcher
- `/ilongrun-model` skill
- 模型管理 helper

`ilongrun-doctor` 也新增了：

- `launcher.ilongrun-model`
- `/ilongrun-model` skill
- 模型管理 helper

建议升级后执行：

```bash
command -v ilongrun-model
ilongrun-model
ilongrun-doctor --refresh-model-cache
```
