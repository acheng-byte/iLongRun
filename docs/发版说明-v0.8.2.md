# 发版说明 v0.8.2 — `ilongrun-model` Tab 页签模式（全局默认 / run模型 / coding模型）

## 概述

`v0.8.2` 继续打磨 `ilongrun-model` 的交互体验：

- 交互式 TUI 现在支持 `全局默认 / run模型 / coding模型` 三个 Tab 页签
- 可以在终端里直接决定这次选模是同时改 run/coding，还是只改其中一个子命令的默认模型
- 命令行直设 `<model>` / `show` / `reset` 仍保持全局语义，不引入新的公开 scope 参数

这次更新依然不改变 `--model` 的全链路锁模语义，也不改变 review / final audit 固定模型策略。

## 新增

### 1. `ilongrun-model` 交互式 TUI 新增 Tab 页签

直接运行：

```bash
ilongrun-model
```

在 TTY 终端里会进入带 Tab 的交互式模型选择器，页签固定为：

- `全局默认`
- `run模型`
- `coding模型`

你可以使用：

- `Tab`：切到下一个页签
- `Shift+Tab` 或左右方向键：反向切页
- `Enter`：按当前页签写入模型
- `r`：弹出确认后执行**全局 reset**
- `Esc` / `q`：取消退出

### 2. TUI 支持浅作用域切换

三个页签的写入语义分别是：

- `全局默认`
  - 同时更新 `ilongrun` 与 `ilongrun-coding`
  - 同时更新全局主执行角色模板（沿用既有逻辑）
- `run模型`
  - 只更新 `commandDefaults.run`
  - 只更新 `skillDefaults.ilongrun`
- `coding模型`
  - 只更新 `commandDefaults.coding`
  - 只更新 `skillDefaults.ilongrun-coding`

## 增强

### 1. reset 语义继续保持全局一致

无论你当前停在哪个页签：

- 在 TUI 里按 `r`
- 或直接执行 `ilongrun-model reset`

都会恢复全局默认模板：

- `run=claude-sonnet-4.6`
- `coding=claude-opus-4.6`

并继续沿用既有的全局 reset 角色恢复逻辑。

### 2. 命令行直设 `<model>` 继续保持全局模式

例如：

```bash
ilongrun-model claude-haiku-4.5
```

这条命令不会区分 run/coding 页签，而是继续按全局默认模式执行：

- 同时影响后续新启动的 `ilongrun`
- 同时影响后续新启动的 `ilongrun-coding`

也就是说：

- **Tab 分页作用域只存在于交互式 TUI**
- **命令行直设 `<model>` 仍是全局模式**

### 3. 审查与终审模型策略保持不变

以下字段继续不会被 `ilongrun-model` 改写：

- `codingAuditModel`
- `roleModels.final-audit-reviewer`
- review 固定角色
- `prompt / resume / status / doctor`
- `fallback / backoff / aliases / displayNames`

## 说明

### 为什么 run/coding 页签不改 `roleModels`？

这是本次明确采用的**浅作用域**策略。

原因是当前共享的 `roleModels` 并不是天然按 run/coding 完全分家的；例如 `executor` 等角色会在不同上下文里复用。

因此 `v0.8.2` 的约束是：

- run/coding 页签只改对应子命令默认模型
- 不在本版本里扩配置协议
- 不拆共享角色真值

如果后续要把 run/coding 的共享角色模板彻底拆开，再单独做协议升级版本。

## 安装升级

```bash

```

安装后建议验证：

```bash
command -v ilongrun-model
ilongrun-model
python3 scripts/lint_ilongrun_skills.py
python3 scripts/selftest_ilongrun.py
bash scripts/ilongrun-doctor --refresh-model-cache
```
