# 发版说明 v0.7.1 — 历史命名硬清理、模型全链路锁定与文档重构

## 这次版本做了什么

`v0.7.1` 是一个**收敛型补丁版本**，重点不是新增命令，而是把多版本迭代后产生的命名漂移、模型语义漂移、文档理解成本一次性收拢。

## 核心变化

### 1. 历史命名硬清理

- 删除 `skills/ilongrun-coding/phase-ship.md`
- 正式拆分为：
  - `skills/ilongrun-coding/phase-audit.md`
  - `skills/ilongrun-coding/phase-finalize.md`
- 终审 agent 改名为：
  - `agents/ilongrun-final-audit-reviewer.agent.md`
- 最终审查报告路径统一为：
  - `reviews/final-review.md`

> 旧命名不再作为当前接口保留；仅在历史发版说明和清理脚本中保留为历史记录或卸载用途。

### 2. `--model` 升级为全链路强制

当显式传入：

```bash
copilot-ilongrun coding --model claude-haiku-4.5 "..."
```

现在会强制作用于整条链路：

- `selectedModel`
- `codingAuditModel`
- `mission.modelAllocation`
- review / audit / finalize
- scheduler 的模型元数据

并且：

- 禁止跨模型 fallback
- 只允许同模型重试

### 3. 仓库命令优先读取仓库配置

直接运行仓库里的：

- `scripts/copilot-ilongrun`

现在会优先读取当前仓库 `config/`，避免被 `~/.copilot-ilongrun/config/` 的旧安装配置污染验证结果。

### 4. README 面向新手重构

主 README 改为新手上手页，重点回答：

- 这是什么
- 适合谁
- 怎么安装
- 第一次怎么用
- 常见问题
- 运行后看哪些文件

并新增：

- README 简化架构图
- `docs/架构与运行机制.md` 详细生命周期图

## 不变的部分

- **没有新增 CLI 命令**
- `ilongrun / ilongrun-coding / ilongrun-status / ilongrun-resume / ilongrun-doctor / copilot-ilongrun` 保持不变
- `config/coding-protocol.jsonc` 仍保持 `0.7.0` 协议版本；本次是 **产品补丁发版 `0.7.1`**，不是协议代际升级

## 升级影响

如果你有自定义脚本或外部流程直接引用以下旧路径，请改为新名字：

- `phase-ship.md` → `phase-audit.md` / `phase-finalize.md`
- `ilongrun-gpt54-audit-reviewer.agent.md` → `ilongrun-final-audit-reviewer.agent.md`
- `reviews/gpt54-final-review.md` → `reviews/final-review.md`

本次发版前已完成：

- `python3 -m compileall scripts`
- `python3 scripts/lint_ilongrun_skills.py`
- `python3 scripts/selftest_ilongrun.py`
- 真实命令验证：
  - `copilot-ilongrun run --model claude-haiku-4.5`
  - `copilot-ilongrun coding --model claude-haiku-4.5`

验证结果确认：

- `modelControlMode = explicit-model-locked`

- 最终审查路径为 `reviews/final-review.md`
