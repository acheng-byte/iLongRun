# iLongRun 快速开始

## 1. 一键安装

```bash
curl -fsSL https://raw.githubusercontent.com/izscc/iLongRun/main/install.sh | bash
```

## 2. 检查环境

```bash
command -v ilongrun
command -v ilongrun-coding
ilongrun-doctor --refresh-model-cache
ilongrun-doctor --notify-test
```

> 如果你在 v0.3.0 首发阶段已经安装过一次，重新执行一遍安装命令即可补齐 `ilongrun-coding`。
> `--notify-test` 仅在 macOS 上有意义，用来检查系统提醒链路是否正常。

## 3. 最常用的几个命令

```bash
ilongrun "修复登录流程并补充测试"
ilongrun-coding "实现登录模块并补测试，最后做 GPT-5.4 终审"
ilongrun-prompt "调研 3 个 AI Agent 编排方案并输出中文结论"
ilongrun-status latest
ilongrun-resume latest
```

其中：
- `ilongrun`：通用入口，自动推断 profile
- `ilongrun-coding`：显式 coding 入口，强制 `profile=coding`

## 4. 运行目录在哪里

每次运行都会在当前工作区生成：

```text
.copilot-ilongrun/
```

其中最重要的是：
- `mission.md`
- `strategy.md`
- `plan.md`
- `scheduler.json`
- `task-list-N.md`
- `workstreams/ws-*/`
- `reviews/gpt54-final-review.md`（coding 任务）
- `reviews/adjudication.md`（coding 任务）

## 5. 什么时候会用到 `/fleet`

只有满足下面条件才会自动尝试：
- 2 到 4 个相互独立的 workstream
- 写集不冲突或已显式分区
- 不依赖中途人工裁决
- 失败后可独立重试

若本机 Copilot CLI 不支持 `/fleet`，iLongRun 会自动降级为 `internal`。
