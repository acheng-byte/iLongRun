# iLongRun 快速开始

这份文档是 README 的展开版，面向第一次上手的人，重点讲：装完之后先做什么、常用命令怎么理解、`--model` 现在到底如何生效。

## 1. 安装

```bash
curl -fsSL https://raw.githubusercontent.com/izscc/iLongRun/main/install.sh | bash
```

## 2. 安装后第一件事：做体检

```bash
command -v ilongrun
command -v ilongrun-coding
command -v ilongrun-model
ilongrun-doctor --refresh-model-cache
ilongrun-doctor --notify-test
```

`ilongrun-doctor` 会检查：

- 命令入口是否完整
- Copilot CLI 是否存在
- 是否已经登录
- 模型缓存是否能刷新
- `/fleet` 是否可探测
- coding protocol / playbooks / review agents / skill lint 是否齐全

## 3. 先从最简单的 run 开始

```bash
ilongrun "帮我阅读当前项目并整理一个 README 修改建议"
ilongrun-status latest
```

这条路径最适合第一次确认：

- 命令是否能启动
- run 目录是否能生成
- 状态看板是否能正常读取

## 4. coding 任务怎么跑

```bash
ilongrun-coding "实现一个小功能，补测试，并完成 review gate"
ilongrun-status latest
ilongrun-resume latest
```

coding 任务固定走这条生命周期：

```text
phase-define
→ phase-plan
→ phase-build
→ phase-verify
→ phase-review
→ phase-audit
→ phase-finalize
```

## 5. `--model` 现在怎么生效

这是这次最重要的行为变化之一。

当你显式传：

```bash
ilongrun-coding --model claude-haiku-4.5 "修一个小 bug"
```

系统会把它视为：

> **全链路强制模型锁定**

覆盖范围包括：

- launcher / supervisor 选择模型
- `scheduler.json` 里的 `selectedModel`
- `codingAuditModel`
- `mission.modelAllocation`
- review gate / audit gate 的 owner model
- 最终报告中的模型元数据

### 这意味着什么？

- 会一直用 `claude-haiku-4.5`
- 不会在中途偷偷切到 `gpt-5.4` 或其他模型
- 不会再出现“入口是 Haiku，终审却换成别的模型”的情况

## 6. `ilongrun-model` 怎么用

如果你想切换**后续新任务**的默认主模型，而不是只锁定某一次 run，可以用：

```bash
ilongrun-model
ilongrun-model claude-haiku-4.5
ilongrun-model reset
```

它的边界是：

- 只影响后续新启动的 `ilongrun` / `ilongrun-coding`
- 不回写当前活动 run 的 `scheduler.json`
- `resume` 仍优先继承目标 run 已有的 `selectedModel`
- review roles 与 final audit 仍保持独立审查模板（默认 `gpt-5.4`）

## 7. 运行目录怎么看

每次 run 都会在当前目录下生成：

```text
.copilot-ilongrun/
```

重点看这些文件：

- `scheduler.json`：系统真值
- `mission.md`：任务目标
- `strategy.md`：总体策略
- `plan.md`：阶段规划
- `task-list-N.md`：阶段清单
- `workstreams/ws-*/status.json`：每个子任务状态
- `reviews/final-review.md`：最终终审报告
- `reviews/adjudication.md`：裁决报告
- `COMPLETION.md`：完成态总结

## 8. 最常用命令说明

### `ilongrun`

通用入口。适合“帮我做这件事”这类长跑任务。

### `ilongrun-coding`

coding 专用入口。适合你明确知道这是开发/修 bug/补测试/评审闭环任务。

### `ilongrun-model`

查看或切换后续新任务的默认主模型模板。

### `ilongrun-status latest`

查看最近一次 run 的状态，不会改文件。

### `ilongrun-resume latest`

继续上一次 run。

### `ilongrun-prompt`

只想先看策略，不想完整开跑时使用。

### `ilongrun-doctor`

安装完必跑；出问题时也优先跑。

## 9. 建议的新手体验顺序

```bash
ilongrun-doctor --refresh-model-cache
ilongrun-model
ilongrun "帮我阅读当前仓库并给出结构建议"
ilongrun-status latest
ilongrun-coding --model claude-haiku-4.5 "实现一个最小改动并补测试"
ilongrun-status latest
```

## 10. 常见排障

### 找不到命令

把下面这一行加入你的 shell 配置：

```bash
export PATH="$HOME/.local/bin:$PATH"
```

### doctor 提示未登录

先执行：

```bash
copilot login
```

然后重新运行：

```bash
ilongrun-doctor --refresh-model-cache
```

### 状态目录存在，但看不懂

先看：

1. `scheduler.json`
2. `ilongrun-status latest`
3. `reviews/final-review.md`（coding 任务）
4. `COMPLETION.md`
