# iLongRun 快速开始

## 1. 一键安装

```bash
curl -fsSL https://raw.githubusercontent.com/izscc/iLongRun/main/install.sh | bash
```

## 2. 环境检查

```bash
command -v ilongrun
command -v ilongrun-coding
ilongrun-doctor --refresh-model-cache
ilongrun-doctor --notify-test
```

doctor 会重点检查：

- 命令入口是否完整
- Copilot 是否已登录
- 默认模型缓存是否正常
- `/fleet` 能力是否可探测
- coding protocol bundle 是否完整
  - `coding-protocol.jsonc`
  - `vendor/agent-skills/`
  - `skills/ilongrun-coding/*`
  - 专项 review agents

## 3. 最常用命令

```bash
ilongrun "修复登录流程并补充测试"
ilongrun-coding "实现登录模块并补测试，完成 review gate 与最终终审"
ilongrun-prompt "调研 3 个 AI Agent 编排方案并输出中文结论"
ilongrun-status latest
ilongrun-resume latest
```

其中：

- `ilongrun`：自动推断 profile
- `ilongrun-coding`：强制 `profile=coding`

## 4. v0.6.0 的 coding run 会长什么样

新的 coding 生命周期固定为：

```text
phase-define
→ phase-plan
→ phase-build
→ phase-verify
→ phase-review
→ phase-audit
→ phase-finalize
```

你会在状态板和 `scheduler.json` 里看到：

- `codingProtocol`
- `swarmPolicy`
- `dependencyGraph`
- `reviewMatrix`

以及每个 workstream 的：

- `swarmMode`
- `writeSet`
- `handoffArtifacts`
- `entryCriteria`
- `exitCriteria`
- `verificationClass`
- `reviewRequired`

## 5. 运行目录在哪里

每次运行都会在工作区生成：

```text
.copilot-ilongrun/
```

关键文件：

- `mission.md`
- `strategy.md`
- `plan.md`
- `scheduler.json`
- `task-list-N.md`
- `workstreams/ws-*/`
- `reviews/gpt54-final-review.md`
- `reviews/adjudication.md`
- `reviews/delivery-audit.md`
- `COMPLETION.md`

## 6. 默认模型和协议配置在哪里改

模型策略：

```bash
~/.copilot-ilongrun/config/model-policy.jsonc
```

coding 协议：

```bash
~/.copilot-ilongrun/config/coding-protocol.jsonc
```

默认规则：

- `ilongrun` / `status` / `prompt` / `resume` / `doctor` → `claude-sonnet-4.6`
- `ilongrun-coding` → `claude-opus-4.6`
- coding 最终终审 → `gpt-5.4`

优先级：

```text
--model > commandDefaults > skillDefaults > roleModels > fallback
```

## 7. `/fleet` 什么时候会用到

只有满足下面条件才会自动尝试：

- 当前 wave 允许外部分发
- workstreams 彼此不互相依赖
- 写集不冲突
- 降级回 internal 后仍可继续

对于 v0.6.0 的 coding run：

- 仅 `phase-build` 允许评估 `/fleet`
- `review / audit / finalize / git / release` 一律 internal

## 8. 建议的第一次体验顺序

```bash
ilongrun-doctor --refresh-model-cache
ilongrun-coding "实现一个 TypeScript 小模块，补测试，并完成 review gate"
ilongrun-status latest
```
