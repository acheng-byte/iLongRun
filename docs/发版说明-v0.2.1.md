# 发版说明 v0.2.1 — 中文状态看板

> 发版日期：2026-04-06

## 概述

v0.2.1 是一个聚焦体验层的快速迭代版本，目标非常明确：**把 `ilongrun-status` 的默认终端呈现从英文描述为主，收敛为中文描述为主**。

这一版不改 scheduler 真值结构，不改命令入口，不改运行目录协议；只增强状态查看时的中文可读性、终端亲和度与用户体感。

## 这次改了什么

### 中文化：状态看板默认说中文

`ilongrun-status` 现在默认要求：
- section 标题使用中文
- 字段标签使用中文
- 状态值使用中文
- backend / mode / profile / control mode / verdict 使用中文
- 需要保留机器真值时，再在中文后附原始值

例如：
- `进行中（running）`
- `直连模式（direct-lane）`
- `内部执行（internal）`
- `可继续（resumable）`

### 统一：补齐展示映射规则

新增并固化以下展示映射：
- status
- backend
- orchestration mode
- profile
- model control mode
- verdict
- 内置 phase 名称

这样可以减少同一命令在不同 run 上出现“部分中文、部分英文”的割裂感。

### 看板示例：完整改成中文版本

`skills/ilongrun-status/SKILL.md` 中的 office profile 示例已经完整中文化，便于模型直接遵循，降低再次回退到英文板式的概率。

## 升级方法

```bash
curl -fsSL https://raw.githubusercontent.com/izscc/iLongRun/main/install.sh | bash
```

如果你已经安装过旧版本，重新执行安装命令即可覆盖升级。

## 兼容性

- 与 v0.2.0 完全兼容
- 不修改 `.copilot-ilongrun/` 运行目录结构
- 不修改 `scheduler.json` 字段协议
- 不修改 `ilongrun / ilongrun-status / ilongrun-resume` 命令入口

## 本版未纳入

- **macOS 系统提醒**：当前仓库内未接入 `osascript`、`terminal-notifier` 或其他系统通知链路，因此 v0.2.1 不包含 Mac 系统提醒能力
- **状态看板脚本化渲染**：本版仍以 skill 输出约束为主，尚未引入独立 TUI / ANSI 渲染器

## 适用场景

如果你当前最在意的是：
- `ilongrun-status` 看起来更像中文产品，而不是英文提示堆叠
- 状态概览更适合中文终端阅读
- 给非技术或中文优先用户展示时更自然

那么 v0.2.1 就是针对这个问题的直接修复版本。
