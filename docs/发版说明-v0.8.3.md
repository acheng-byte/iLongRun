# 发版说明 v0.8.3 — `ilongrun-model show` 品牌化看板与 TUI 稳定性修复

## 概述

`v0.8.3` 是对 `ilongrun-model` 的一次体验收口：

- `ilongrun-model show` 的文本输出现在统一为 iLongRun 品牌化中文看板
- 交互式选模器补上了更稳健的终端尺寸容错
- 保持 `--json` 不变，方便脚本化消费

这次更新不改变 `global / run / coding` 三页签的写入语义，也不改变 review / final audit 的固定模型策略。

## 新增

### 1. `show` 输出升级为品牌化终端看板

现在执行：

```bash
ilongrun-model show
```

会看到与 `ilongrun-doctor` / `ilongrun-status` / 安装看板同风格的终端界面，包括：

- 顶部品牌标题
- 当前模式、写入作用域、run/coding 模板摘要
- 写入目标路径区块
- 当前边界说明
- 使用提示与下一步命令

## 增强

### 1. 小终端尺寸下的 TUI 容错增强

修复了交互式 `curses` 界面在某些终端尺寸下可能报错：

- `addnwstr() returned ERR`

现在会对屏幕边界进行保守写入，尽量避免因为窗口太小而直接失败。

### 2. 文本模式与脚本模式语义更清晰

- `ilongrun-model show`：面向人类阅读的品牌化看板
- `ilongrun-model --json show`：面向脚本处理的稳定 JSON

也就是说：

- **人看：默认文本看板**
- **脚本读：明确使用 `--json`**

## 说明

### 关于本次验证方式

这次修复以**本地脚本级验证**为主，不主动触发额外的 Copilot CLI 配额消耗型调用。

重点确认的是：

- `manage_ilongrun_model.py` 的 Python 语法正确
- 非交互 `show` 路径能正常输出品牌化看板
- 已安装 helper 能被直接同步更新

## 安装升级

如果你已经装过 iLongRun，但只想快速拿到这次修复，也可以直接同步本地 helper / launcher。

如果要完整升级，仍可在你方便的时候重装：

```bash
curl -fsSL https://raw.githubusercontent.com/izscc/iLongRun/main/install.sh | bash
```
