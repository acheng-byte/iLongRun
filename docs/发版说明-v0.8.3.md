# 发版说明 v0.8.3 — `ilongrun-model` 完整整合版（Tab 页签选模 + 品牌化看板 + 稳定性修复）

## 概述

`v0.8.3` 是 `ilongrun-model` 这一轮能力演进的**完整整合版发行说明**。

如果你之前没有仔细看 `v0.8.2`，只看这一篇也可以直接理解当前版本的全部能力。

当前 `ilongrun-model` 已经具备：

- 终端裸命令的交互式模型选择器
- `全局默认 / run模型 / coding模型` 三页签 Tab 作用域切换
- `show / <model> / reset / --refresh` 命令入口
- 品牌化中文终端看板输出
- 更稳健的小终端尺寸 TUI 渲染容错

它的职责始终保持不变：

> **只修改 iLongRun 后续新任务的默认模板，不改当前 Copilot CLI 会话的原生 `/model`，也不回写正在运行中的 run。**

## 新增

### 1. `ilongrun-model` 支持交互式 Tab 页签选模

直接运行：

```bash
ilongrun-model
```

在 TTY 终端里会进入交互式模型选择器，并提供 3 个固定页签：

- `全局默认`
- `run模型`
- `coding模型`

快捷键：

- `Tab`：切到下一个页签
- `Shift+Tab`：切回上一个页签
- `← / →`：左右切页
- `↑ / ↓` 或 `j / k`：上下移动
- `Enter`：按当前页签写入模型
- `r`：弹确认后执行**全局 reset**
- `Esc / q`：取消退出
- `Backspace`：删除搜索词

### 2. 交互式 TUI 作用域语义明确

#### `全局默认`

用于同时切换：

- `ilongrun`
- `ilongrun-coding`

会更新：

- `commandDefaults.run`
- `commandDefaults.coding`
- `skillDefaults.ilongrun`
- `skillDefaults.ilongrun-coding`
- 全局主执行角色模板（沿用既有主执行角色逻辑）

#### `run模型`

用于只切换 `ilongrun` 默认模型。

只会更新：

- `commandDefaults.run`
- `skillDefaults.ilongrun`

不会改：

- `commandDefaults.coding`
- `skillDefaults.ilongrun-coding`
- 任意 `roleModels`

#### `coding模型`

用于只切换 `ilongrun-coding` 默认模型。

只会更新：

- `commandDefaults.coding`
- `skillDefaults.ilongrun-coding`

不会改：

- `commandDefaults.run`
- `skillDefaults.ilongrun`
- 任意 `roleModels`

### 3. `show / <model> / reset / --refresh` 保持公开 CLI 入口

你现在可以使用：

```bash
ilongrun-model
ilongrun-model show
ilongrun-model --refresh
ilongrun-model claude-haiku-4.5
ilongrun-model reset
```

它们的语义分别是：

- `ilongrun-model`
  - TTY：进入交互式 TUI
  - 非 TTY：退化为 `show`
- `ilongrun-model show`
  - 查看当前默认模板摘要
- `ilongrun-model --refresh`
  - 先刷新模型缓存，再进入 TUI / show
- `ilongrun-model <model>`
  - **命令行直设，始终按全局模式写入**
- `ilongrun-model reset`
  - **全局 reset**，恢复默认模板

## 增强

### 1. `show` 输出升级为 iLongRun 品牌化中文看板

现在执行：

```bash
ilongrun-model show
```

会输出统一的 iLongRun 风格终端看板，而不是简单的纯文本列表。

看板会包含：

- 当前模式
- 写入作用域
- `run` / `coding` 当前模板
- 审查模板摘要
- 写入目标路径
- 当前边界说明
- 使用提示

这让它和以下命令的视觉风格保持一致：

- `ilongrun-doctor`
- `ilongrun-status`
- 安装完成看板

### 2. TUI 渲染稳定性增强

修复了某些终端尺寸下交互式选模器可能报错的问题：

- `addnwstr() returned ERR`

现在会在 `curses` 写入阶段进行更保守的边界保护，尽量避免因为窗口偏小而直接退回到 `show`。

### 3. 文本模式与脚本模式分工更清晰

当前推荐这样使用：

- **给人看**：

```bash
ilongrun-model show
```

- **给脚本读**：

```bash
ilongrun-model --json show
```

也就是说：

- 默认文本输出面向人类阅读
- 机器消费请显式使用 `--json`

## 使用说明

### 1. `ilongrun-model` 改的到底是什么？

它改的是：

> **iLongRun 后续新任务的默认模板**

不是：

- 当前 Copilot CLI 会话的原生 `/model`
- 正在运行中的 run 的 `scheduler.json`
- 已存在 run 的 `selectedModel`

### 2. `resume` 会不会被新默认值打断？

不会。

`resume` 仍优先继承目标 run 已有的 `selectedModel`。

也就是说：

- `ilongrun-model` 改的是**未来默认模板**
- `resume` 继续尊重**历史 run 已选模型**

### 3. 哪些字段不会被 `ilongrun-model` 改写？

以下字段继续保持独立，不跟随 `ilongrun-model` 改动：

- `codingAuditModel`
- `roleModels.final-audit-reviewer`
- review 固定角色
- `prompt / resume / status / doctor`
- `fallback / backoff / aliases / displayNames`

### 4. 为什么 `run模型` / `coding模型` 不改 `roleModels`？

这是本轮明确采用的**浅作用域**策略。

原因是当前共享的 `roleModels` 并不是天然按 run/coding 完全分家的，部分角色会在不同上下文中复用。

因此当前版本的约束是：

- `run模型` / `coding模型` 页签只改对应子命令默认模型
- 不新增配置协议字段
- 不拆共享 `roleModels` 真值

如果未来要把 run/coding 的共享角色模型彻底拆分，再单独做协议升级版本。
