# 发版说明 v0.8.4 — 完成态真值收敛与通用 gate 加固

## 概述

`v0.8.4` 这一版不是围绕某个业务样例做补丁，而是基于一次真实 coding 样本 run 复盘，对 iLongRun 的**通用执行内核**做了一轮加固。

这次发版重点解决的是：

- completed / complete 状态可被手写伪造
- review / finalize 产物不完整但仍可能被账面完成掩盖
- verifier 的 `ok=false` 与 completion score / verdict 语义割裂
- `sessionEnd` 之后账本没有自动落一次 precheck
- doctor 只能看安装，不会直接指出当前工作区 run 健康问题

## 新增

### 1. 当前工作区 Run 健康检查

`ilongrun-doctor` 现在除了检查安装、登录、模型与 bundle，还会额外检查当前工作区 active/latest run 的健康状态，包括：

- top-level `state` / `status` 是否冲突
- `active-run-id` 是否残留在已完成 run 上
- `COMPLETION.md` / `reviews/final-review.md` / `reviews/adjudication.md` 是否缺失
- review workstream 是否出现 placeholder 伪完成
- git 是否错误跟踪了 `.copilot-ilongrun/`、`node_modules/`、`dist/` 等生成态目录

### 2. 真实样本复盘文档与项目级整改说明

新增两份仓库文档：

- `docs/真实样本复盘-test5-game.md`
- `docs/项目全局审计与整改说明.md`

前者记录真实样本的证据化复盘，后者沉淀本轮通用整改原则。

## 增强

### 1. 完成态真值收敛

从这一版开始：

- `scheduler.json` 的 top-level `state` 是 run 完成态唯一真值
- top-level `status` 只保留 legacy 兼容读取
- 新的 ledger 写盘会自动清理 top-level `status`
- 如果 `status` 与 `state` 冲突，verify / doctor 会直接判定 drift

### 2. placeholder review 伪完成识别更严格

`result.md` / `evidence.md` 出现以下情况时，会被识别为伪完成：

- 空文件
- pending 模板
- placeholder 文案
- TODO / TBD / 待补 / 待填写等模板残留

### 3. completion score 与 verdict 语义统一

现在只要 verify 存在 hard failure 或 drift：

- completion score 会降级或封顶
- `deliveryVerdict` 会同步降级
- 不再出现 `ok=false` 但仍然给出 `A / prototype-ready` 的情况

### 4. `sessionEnd` 自动 precheck 落账

`sessionEnd` 事件会自动做一次本地 verification precheck，并把：

- verification state
- recommended action
- failure class
- hook precheck 时间戳

写回账本。

注意：

- 这不是自动 finalize
- 它只负责把当前 run 的真实健康状态落账，方便恢复、复盘和 doctor 诊断

## 修复

### 1. finalize 收尾顺序更安全

`finalize_ilongrun_run.py` 现在会先准备 `COMPLETION.md`，再进入清理 active pointer 的收尾写盘，降低“账本已 complete 但 completion 丢失”的中间不一致窗口。

### 2. audit / review 缺件的阻断时机更合理

`verify` 不再对所有 in-progress coding run 一律强制要求 final review / adjudication。

只有在以下语境中，它们才会成为硬失败：

- run 已进入 audit / finalize 语境
- run 已被标为 complete/completed
- finalize helper 正在执行最终 precheck

这样可以减少对中途运行态的误伤，但不会放松 finalize 门禁。

### 3. completed 但缺 completion 的 run 不再自动清 active 指针

如果一个 run 被标成 complete/completed，但 `COMPLETION.md` 仍缺失，则系统不会自动清掉 `active-run-id`，而是保留指针并把问题显示给 verify / doctor。

## 安装升级

如果你已经安装过旧版本，建议：

```bash
bash install.sh
ilongrun-doctor --refresh-model-cache
```

然后在一个真实工作区里额外执行一次：

```bash
ilongrun-doctor
```

确认新的 **“当前工作区 Run 健康”** 区块能正常显示。

## 说明

这次发版的验证策略默认仍以**非 Copilot CLI** 回归为主：

- `python3 scripts/selftest_ilongrun.py`
- `python3 scripts/lint_ilongrun_skills.py`
- `bash scripts/ilongrun-doctor --refresh-model-cache`
- 定向 verify / finalize / hook_event / ledger sync 场景测试

也就是说，本版的核心内核修复和回归，不依赖真实 Copilot CLI 长跑额度。
